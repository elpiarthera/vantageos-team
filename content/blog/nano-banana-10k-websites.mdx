## Key takeaways

### 1. The animation trick is absurdly simple — 3:00

Every scroll-driven animation on these premium landing pages is just a video broken into frames. Nano Banana 2 generates a start frame and end frame. Cling 3.0 animates between them. ffmpeg extracts individual frames. Claude Code maps each frame to a scroll position. The result looks like a $50k agency build. The process takes 30 minutes.

### 2. Two skills do all the heavy lifting — 6:30

The entire workflow runs on two Claude Code skills: a front-end design skill (modified from Anthropic's official version) and a video-to-website skill. Both are markdown files containing best practices for creating animated, scroll-driven websites. The skills are given away free in the creator's community — the value is not in the files but in knowing how to use them iteratively.

### 3. Plan mode first, always — 12:00

Before building anything, the creator switches Claude Code to plan mode. The agent reads the video, asks clarifying questions (product name, content sections, layout preferences), and produces a detailed implementation plan with brand identity, fonts, colors, and section-by-section build steps. This planning step is why the first iteration comes out clean enough to ship with minor tweaks.

### 4. Context management matters mid-build — 16:30

At 53% context usage, the creator clears the conversation before making iteration requests. This is practical wisdom: context rot degrades output quality. He feeds back only the specific feedback (feature two appears too late on scroll) rather than re-explaining the entire project. Targeted, lean feedback loops produce better iterations than long conversations.

### 5. The deployment gotcha: frames in .gitignore — 18:45

When pushing to GitHub, the animation frames were excluded (likely by a default .gitignore pattern for media files). The deployed site rendered everything except the actual animation. The fix was simple — include the frames in the repo and redeploy. A useful reminder: always verify that your production deploy matches localhost, especially with large media assets.

### 6. The business model is the real takeaway — 20:30

The technical pipeline enables a specific business model: find businesses with bad websites, build a demo landing page with their product in one day, show it to them, charge $5k-$10k plus monthly hosting and maintenance. This undercuts traditional agencies on both speed and cost while producing visually comparable results. The recurring revenue from hosting and updates makes it sustainable.
