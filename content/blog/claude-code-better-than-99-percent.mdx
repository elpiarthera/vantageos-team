## Key takeaways

### 1. One input, many outputs — that's the leverage equation

Mark's YouTube command center captures it: from one topic input, the system generates 15 thumbnails, face-swapped images, thumbnail briefs, editing briefs, script outlines, SEO tags, titles, and descriptions. The same principle drives the Instagram system they build live: say "make me viral," provide a topic, and get 20 ranked hooks plus captions. This is what leverage actually looks like — one decision, multiple artifacts.

### 2. CLAUDE.md is your persistent brain

Every mistake, every workflow, every actor preference gets saved to CLAUDE.md. New sessions start smart instead of starting blind. Mark makes the comparison explicit: in ChatGPT, if you don't document a failure, it keeps happening. In Claude Code, you tell it to learn from the mistake and commit it to memory. The knowledge base compounds over time.

### 3. Apify MCP eliminates manual scraper shopping

Instead of browsing the Apify marketplace, testing actors manually, and checking success rates, the MCP does it all. It searches actors, ranks them by usage and success rate, tests them, and reports back. When one actor fails, it automatically finds alternatives. Mark shows this live — the MCP discovers that transcript scraping needs a different actor than post scraping, and it finds the right one without human intervention.

### 4. SQLite as free local database — stop paying for prototypes

Every vibe-coded app that uses Supabase costs $10 to spin up. SQLite runs locally, costs nothing, and handles prototyping perfectly. Build the system, iterate until it works, then upgrade to a hosted database if you need to. Mark treats this as non-negotiable for early-stage builds.

### 5. The caption vs. transcript bug — why human-in-the-loop matters

The system was working. Hooks were being generated. Everything looked fine — until Brock noticed the hooks all said "Comment X and I'll send you Y." The system had stored Instagram captions in the transcript column and was training on the wrong data. No automated check caught this. A human watching the output did. Mark is direct about it: "You can't just go off to the beach and come back and expect good results."

### 6. Claude Co-Work gives you 30-40% of Claude Code's power

Mark's honest assessment of Co-Work: it's the gateway drug, not the destination. It covers the most common non-technical use cases with one-click buttons. But it's vibe-coded, full of bugs, and nowhere near the full firepower of Claude Code in a terminal. Use it to get comfortable. Then upgrade.
