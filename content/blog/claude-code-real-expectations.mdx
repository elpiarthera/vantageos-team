## Key takeaways

### 1. Claude Code is a script factory, not a chatbot

The fundamental insight: instead of asking an AI to reason through a task every time (with hallucination risk at every step), Claude Code writes a script, tests it, validates it, and then runs that deterministic code on demand. The reliability difference between "ask AI to figure it out" and "run the code AI already wrote and tested" is enormous.

### 2. The hardest part is always data extraction

Every workflow hit the same wall: getting data out of existing software. QuickBooks has no straightforward API for direct pulls. Estimating software varies wildly. The presenter had to build a Zapier-to-Airtable bridge just to get accounting data into a format Claude Code could read. This is the real work -- and it is where most non-technical users give up.

### 3. Plan mode as default workflow

The presenter defaults to plan mode: discuss what you want, Claude creates a plan, you approve, then it executes. This is not a beginner setting -- it is the correct production workflow. It gives you a checkpoint before anything runs, which matters when Claude has write access to your files.

### 4. MCP connectors unlock real business automation

Connecting Claude Code to Gmail, Google Drive, Notion, Airtable, and Canva through MCP turns it from a coding tool into a business automation platform. The email triage workflow -- scanning for outstanding actions, RFIs, submittals -- runs entirely through the Gmail connector without leaving Claude Code.

### 5. Tender extraction: from inbox to CRM in minutes

The third workflow pulls tender documents from email, extracts structured data, and writes it to an Airtable CRM. The presenter built it from a single natural-language prompt. Setup time: roughly 20-30 minutes of iteration. The demo makes the critical point that you do not need to understand Python to use what Claude Code produces.

### 6. Safety matters when Claude has file access

Claude Code can read, write, edit, and delete files in its working folder. The presenter recommends three mitigation strategies: use "Ask Permissions" mode for folders with important documents, work on duplicates rather than originals, and use plan mode to review before execution.

## How we applied this

The Zapier-as-bridge pattern -- routing data from API-limited software through an intermediary into a format Claude Code can consume -- is now a standard pattern in our client delivery process. We also adopted the presenter's "plan first, always" approach as a non-negotiable in our own agent workflows. When building automations for clients whose tools lack APIs, we document the workaround chain upfront instead of discovering it mid-build.
