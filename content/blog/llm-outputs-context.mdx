## Key takeaways

### 1. Context degradation is the real problem — 0:00

The video opens with the core insight most AI users feel but cannot name: output quality degrades the longer you chat. Every new message adds to the context window, diluting the instructions you gave at the start. Skills solve this by loading focused instructions only when needed, keeping the context lean.

### 2. Progressive disclosure is the architecture — 2:15

Claude Skills use progressive disclosure — loading information in stages rather than consuming your entire context upfront. Think of it as a cookbook where Claude only opens the page it needs. This is not just a performance optimization. It is the reason skills produce more consistent output than pasting the same instructions repeatedly.

### 3. Projects vs. Skills: know which to use — 5:30

Projects are persistent context containers. Skills are executable capabilities. The practical framework: if Claude needs to know something for every conversation (brand voice, ICP), that is a project. If you need a repeatable workflow with clear inputs, steps, and outputs, that is a skill. Most people confuse these. Getting it right saves thousands of tokens.

### 4. Skill anatomy is simpler than you think — 8:00

Every skill starts with a skill.md file — a markdown file with YAML frontmatter defining the name and description, followed by step-by-step instructions. Additional resource folders (logos, fonts, examples, writing style) go alongside it. The key principle: only add context Claude does not already have. Lean beats comprehensive.

### 5. Self-improving skills are the real power — 14:30

After each use, you tell Claude what worked and what did not, then ask it to update the skill.md. Every build makes the skill better. This iterative refinement loop is the difference between a static prompt and a living workflow that compounds in quality over time.

### 6. Custom skills are still individual — 6:45

An important limitation: custom skills are not shared organization-wide on team or enterprise plans. Each user maintains their own. For teams that need standardized AI workflows across people, this is a gap — and exactly the kind of problem a shared skill library solves.
